# Patch System

## Concept
Patch keys are collectible fragments of code or compiled modules that the player can use to stabilise corrupted areas. They serve both as progression gates and narrative artefacts, representing developer efforts to fix broken builds.

## Types of Patch Keys
- **Stability Patches**: Restore geometry, textures and basic functionality of a room.
- **Feature Patches**: Unlock or repair specific mechanics like a broken elevator or disabled console.
- **Security Patches**: Override corrupted locks and firewalls, opening new wings or secret labs.
- **Experimental Patches**: Introduce temporary cheat-like effects (e.g. noclip, invincibility) that help solve meta puzzles but may also increase instability elsewhere.

## Acquisition
Players earn patch keys by:
- Completing glitch-lens puzzles that require variable manipulation.
- Extracting glitch energy from unstable artefacts, converting it into a patch.
- Solving meta challenges such as decoding developer notes, exploring ARG clues, or reconstructing missing source files.

## Application
Patch terminals and consoles are scattered around the museum. When a player applies a patch, the environment reconfigures in real time: textures revert, collapsed floors rebuild, and NPCs regain sanity. Each patch consumes one key, requiring strategic consideration about when and where to use them.

## Narrative Integration
Each patch key includes metadata that reveals a snippet of in-world lore: an email between developers, a git commit message, or a bug ticket. Collecting and reading these not only gives hints for puzzles but also pieces together the larger story of why the museum build was abandoned.
