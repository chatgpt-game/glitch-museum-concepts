# Glitch Lens Mechanics

## Overview
The Glitch Lens is the player's primary interface for interacting with the museum's anomalies. This experimental handheld device overlays debug tools and reveals secrets hidden beneath corrupted code.

## Core Functions
- **Reveal Hidden Data**: When activated, the lens displays developer comments, metadata tags, and debug variables embedded within objects, allowing players to piece together the backstory of each exhibit.
- **Alternate Realities**: Some objects exist in multiple states. Peering through the lens shifts the player's perspective, letting them see an object's original, corrupted, and patched versions. This is crucial for solving puzzles.
- **Manipulate Properties**: The lens can temporarily modify simple variables on tagged objects (e.g., gravity, transparency, playback speed), causing doors to open or platforms to move. These interactions often require solving logic puzzles or matching code snippets.
- **Glitch Extraction**: By focusing on shimmering artifacts, the lens can extract unstable "glitch energy." This is converted into **patch keys**, which are then used to stabilize rooms.
- **Signal Analysis**: Some audio and visual signals appear as noise until viewed through the lens. Decoding them reveals passwords, cipher keys, or hidden AR content.

## User Interface
The lens appears as an overlay with a grid and scanning reticle. A **stability meter** at the edge indicates how much corruption the lens can handle before it requires recalibration. Icons display available interactions, such as variable fields that can be edited, and the number of patch keys collected.

## Accessibility Considerations
Players can adjust contrast, colorblind filters, and text size for the overlay. An optional auto-translate system converts pseudo-code into plain language for younger or less technical players.

## Glitch Lens & Narrative
Narratively, the Glitch Lens is a prototype tool created by the museum's developers. Logs scattered across the world mention how it was used to debug exhibits. Its upgrade path ties into the story — players restore missing modules to unlock new functions, such as audio spectrogram views or real-time code editing.
